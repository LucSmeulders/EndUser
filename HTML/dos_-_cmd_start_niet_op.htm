DOS - CMD start niet op

Wanneer je op het AXITO icoontje op het bureaublad klikt, gebeurt niets!  De command-prompt (vroeger DOS) via een snelkoppeling op het windows-bureaublad werkt niet goed: Wanneer je een programma opstart via een snelkoppeling, gebeurt eigenlijk niets! Het programma geraakt niet opgestart.

 
Opmerking: Wanneer je echter eerst CMD.EXE opstart via "uitvoeren" en daarna bijv. in de map \AXITO\EXE  het commando AXITO ingeeft, start het programma wel goed op!

 

 

Oplossing van dit CMD-probleem:

- Zie o.a. http://techrepublic.com.com/5208-11192-0.html?forumID=52&threadID=178600&messageID=1818452

- De oplossing komt op het volgende neer:
 

0. Start de command-prompt (DOS)

1. Start de Register-editor (Regedt32.exe).
2. Zoek naar en klik vervolgens op de volgende waarde:
 HKEY_CURRENT_USER \  Software \ Microsoft \ Command Processor
3. Klik op EnableExtensions
4. Waardegegevens moet gelijk zijn aan het getal 1
 If it is set to 0, cmd.exe extensions have been disabled. It should be set to 1.
5. Klik op OK
7. Sluit de Register-editor af.
 

 

 
Opmerking: Bovenstaande oplossing is afkomstig van onderstaande tekst:

--> Check the registry for this setting:
 
 Hive: HKEY_CURRENT_USER
 Key: Software\Microsoft\Command Processor
 Name: EnableExtensions
 Type: REG_DWORD
 Value: 1 enable
 
 If it is set to 0, cmd.exe extensions have been disabled. It should be set to 1.
